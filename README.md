# 高级广告拦截检测

[![License: GPL v3](https://img.shields.io/badge/License-GPLv3-blue.svg)](https://www.gnu.org/licenses/gpl-3.0)

## 项目简介

这是一个专为网站站长设计的高级广告拦截检测方案。在当今广告拦截器广泛使用的环境下，许多非广告资源（如网站Logo、统计代码、功能脚本等）常常被误拦截，导致网站功能不完整或显示异常。

本项目提供了一套完整的解决方案，通过多种检测技术准确识别广告拦截器，并以友好的方式引导用户将网站加入白名单，从而保护网站资源的正常加载。

## 功能特性

### 精准检测

· 7种独立检测方法：广告脚本、广告元素、广告资源、广告域名、CSS类名、iframe、定时器差异
· 智能判定逻辑：多方法交叉验证，降低误报率
· 异步检测机制：不影响页面加载性能

### 美观界面

· 现代化弹窗设计：类似SweetAlert的视觉效果
· 响应式布局：完美适配桌面和移动设备
· 平滑动画：优雅的弹出和关闭动画
· 直观图标：红色"✕"图标清晰传达问题

### 用户体验

· 友好引导：避免强制性要求，强调共赢解决方案
· 详细教程：提供完整的广告拦截器关闭步骤
· 智能防骚扰：24小时内只显示一次提示
· 多语言支持：中文界面，易于理解

### 技术优势

· 轻量级代码：压缩后仅几KB大小
· 易于集成：简单的三步集成过程

· 跨浏览器兼容：支持主流现代浏览器

## 检测原理

本项目采用多重检测机制确保准确性：

1. 脚本加载检测：尝试加载Google AdSense脚本
2. 元素隐藏检测：创建模拟广告元素检测是否被隐藏
3. 资源拦截检测：请求已知广告资源检测是否被阻止
4. 域名访问检测：向广告域名发起请求检测连通性
5. CSS类名检测：使用广告相关类名检测样式处理
6. iframe加载检测：尝试加载广告iframe
7. 性能差异检测：利用广告拦截器可能造成的处理延迟

只有当多种方法同时检测到广告拦截器时，才会向用户显示提示，确保极低的误报率。
